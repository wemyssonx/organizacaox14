Crie um aplicativo web completo e funcional usando a plataforma Bubble.io (no-code/low-code), inspirado no estilo de interface do Trello para gerenciamento de equipes de evento. O app se chama "Organizador do 14º Aniversário da Filosofia X no Pará" e serve para cadastrar e organizar participantes em departamentos específicos, com foco em UI/UX intuitiva, concisa e mobile-friendly. Use um layout de boards (como listas verticais ou colunas Kanban) onde cada departamento é uma coluna/lista, e cada participante é um card móvel que pode ser arrastado entre colunas se necessário (para reatribuição). Inclua autenticação simples via e-mail/senha para administradores editarem, e visualização pública para todos verem a organização sem editar.
Estrutura Geral do App:

Página Inicial (Dashboard): Uma visão geral com um board principal estilo Trello. Cada coluna representa um departamento (listados abaixo). Dentro de cada coluna, cards representam participantes, mostrando nome, telefone, e-mail, cidade/estado e função escolhida. Cards devem ser clicáveis para expandir detalhes. Adicione um botão flutuante para "Novo Cadastro" que abre um formulário modal. Inclua filtro por departamento e busca por nome/telefone. Use cores temáticas: azul escuro para fundo, branco para cards, e ícones simples (ex: megafone para Marketing).
Formulário de Cadastro: Um form intuitivo com campos obrigatórios: Nome, Telefone, E-mail, Cidade/Estado. Em seguida, uma seção de seleção de departamento (dropdown com os listados abaixo), e dentro de cada departamento, radio buttons ou dropdown para funções específicas (listadas por departamento). Inclua opção "Outro" para funções customizadas. Ao submeter, o participante é adicionado como card na coluna do departamento escolhido. Integre com Google Sheets/Excel para importação inicial de dados existentes (via API ou upload CSV), mapeando colunas do Excel para os campos (ex: nome na coluna A, telefone na B, etc.).
Funcionalidades Adicionais:

Arrastar e soltar cards entre colunas para reatribuir departamentos.
Edição de cards: Administradores podem editar detalhes ou remover.
Exportação: Botão para exportar o board como PDF ou CSV, organizado por departamento.
Notificações: Envie e-mail automático ao cadastrado confirmando inscrição e função.
Responsividade: Garanta que funcione bem em desktop e mobile, com colunas empilhadas em telas pequenas.
Segurança: Dados armazenados em database do Bubble, com autenticação para edições.

Instruções para Implementação no Bubble:

Use elementos como Repeating Groups para as colunas de departamentos, com cards como Groups dinâmicos.
Database: Crie uma data type "Participante" com fields: nome (text), telefone (text), email (text), cidade_estado (text), departamento (text), funcao (text).
Workflows: Para cadastro, use "Create a new thing" no submit. Para importação, configure um plugin ou API para Google Sheets.
Estilo: Minimalista, com fontes sans-serif, espaçamento amplo e animações suaves no arrastar.
Teste: Inclua dados de exemplo para 5-10 participantes fictícios distribuídos nos departamentos para validar.
